# 아크팬텀(ARC-FTNM) 기여 가이드라인

아크팬텀 프로젝트에 관심을 가져주셔서 감사합니다! 이 프로젝트는 MMIX 아키텍처를 백엔드로, WASM을 프런트엔드로 하는 CPU를 제작하고 발전시키며 시스템 프로그래밍으로도 뻗어나갈 것입니다. 여러분의 기여가 아크팬텀을 더 예술적인 시스템으로 만듭니다.

## 어떻게 기여할 수 있나요?

### 버그 리포트
- 문제를 발견했다면 [Issue] 탭에 상세히 기록해 주세요.
- 실행 환경(OS, 브라우저 버전 등)과 재현 방법을 포함해 주시면 큰 도움이 됩니다.

### 기능 제안 및 개선
- 새로운 아이디어가 있다면 언제든 Issue를 통해 제안해 주세요.
- MMIX 명령어 집합 구현이나 Rust 기반 커널 최적화 등 기술적인 논의를 환영합니다.

### 풀 리퀘스트(Pull Request) 절차
1. 리포지토리를 포크(Fork)합니다.
2. 새로운 브랜치를 생성합니다. (`feature/멋진기능` 또는 `fix/버그수정`)
3. 코드를 작성하고 테스트를 수행합니다.
4. 변경 사항을 커밋하고 푸시합니다.
5. 원본 리포지토리의 `main` 브랜치로 Pull Request를 보냅니다.

## 코딩 컨벤션
- **Rust:** 기본적으로 `rustfmt`를 사용하여 코드를 정리합니다.
- **문서화:** 모든 공개(public) 함수와 모듈에는 주석을 달아주세요.
- **테스트:** 새로운 기능을 추가할 때는 반드시 단위 테스트를 포함해 주세요.
